# AI-Stock 功能头脑风暴（第三轮）— 用户真实使用视角

**日期**: 2026-04-01
**方法论**: brainstorm-ideas-new（PM / 设计师 / 工程师 三视角）
**特别角度**: 模拟"用了3个月的真实用户"，找出产品使用中的摩擦点和断层

---

## 核心发现：「洞见到行动的断层」

用户使用轨迹分析：
- 第1天：生成分析，很满意
- 第14天：AI推荐了股票，但不知道现在价格如何
- 第30天：有了很多分析，但不知道"今天该做什么"

**根本问题**：产品能帮用户产生洞见，但三个模块各自独立，用户无法从"分析"自然流向"行动"。

---

## 产品经理视角（填补断层）

| # | 功能 | 解决痛点 |
|---|------|---------|
| PM1 | 决策追踪日志 | 记录"因为这篇分析买了XX"，建立决策-结果闭环 |
| PM2 | 分析时效性标签 | 超90天分析自动标"可能已过时"，防止旧分析干扰 |
| PM3 | 今日行动清单 | 每天生成"今日重点"，消除"打开不知道看什么"的迷失感 |
| PM4 | 行业注意力地图 | 可视化研究精力分布，与持仓做对比 |
| PM5 | 分析→观察清单一键转化 | 从AI分析直接把推荐股票加入缠论监控列表 |

## 设计师视角（消除模块割裂感）

| # | 功能 | 解决痛点 |
|---|------|---------|
| D1 | 三模块统一首页 | 今日分析+自选股动态+买卖信号，一屏全览 |
| D2 | 股票行情内嵌卡片 | 分析里的股票名称旁显示当前价格+涨跌幅 |
| D3 | 跨模块关联线索 | 个股K线页显示相关模块一分析，模块三信号显示宏观背景 |
| D4 | 分析置信度自标注 | 保存时打1-5星置信度，复盘看自己思维状态 |
| D5 | 空状态友好设计 | 知识库/自选股为空时，给出具体引导而非空白页 |

## 工程师视角（平台能力补强）

| # | 功能 | 解决痛点 |
|---|------|---------|
| E1 | 行情数据WebSocket实时推送 | 页面顶部显示自选股实时价格条，无需刷新 |
| E2 | K线图本地30天缓存 | 首次加载后缓存，下次打开秒显示 |
| E3 | 知识库老化自动检测 | 定期对比旧分析提到的股票实际走势，生成准确性摘要 |
| E4 | 分析草稿自动保存 | 输入中断时保存草稿，下次继续 |
| E5 | 导出功能 | 导出为PDF/Markdown，消除数据安全顾虑 |
