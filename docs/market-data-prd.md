## 执行摘要

面向A股散户的个股行情展示模块，解决用户在分析过程中需要快速查看股票技术面+基本面+行业对比的需求。

用户核心诉求是：**从模块一分析文章中发现股票 → 点击后立即看到完整行情判断所需的一切信息**，不需要切换到同花顺/东财。

与同花顺/东财相比，本模块的差异化在于：**去广告、去噪音、聚焦散户真正用到的信息、深度联动模块一和模块三**。

---

## 问题陈述

### 用户核心场景
散户在模块一看完AI分析文章后，文章里提到"推荐关注宁德时代（300750）"，用户需要：
1. 立即看到该股K线形态，判断当前处于高位还是低位
2. 快速确认估值是否合理（PE/PB）
3. 了解该股在行业内的地位和横向对比
4. 这一系列动作**不能打断阅读节奏，不能让用户离开当前工具**

### 现有工具的痛点
- 同花顺/东财：广告多、界面复杂、信息过载
- 没有与个人分析日志的关联
- 自选股和策略监控是分离的

## 用户故事 & 验收标准

### Story 1：从模块一跳转到个股K线页

**作为** 正在阅读AI分析文章的散户
**我希望** 点击文章中的股票代码后直接看到该股K线
**从而** 不打断阅读节奏，快速判断是否值得进一步研究

**验收标准：**
- [ ] 模块一文章正文、摘要中出现的股票代码（如"300750"或"宁德时代"）高亮显示，鼠标悬停时出现下划线提示可点击
- [ ] 点击股票代码，在当前页面右侧或弹出抽屉中展示K线详情页（不完全跳页，保留左侧文章内容）
- [ ] 抽屉/侧边栏默认显示：日K线图（含MA5/MA10/MA20 + 成交量 + MACD）
- [ ] 侧边栏顶部显示：股票名称、代码、当前价格、今日涨跌幅、所属申万行业
- [ ] 侧边栏有"展开完整页面"按钮，点击跳转到个股完整详情页

---

### Story 2：个股完整详情页

**作为** 散户
**我希望** 在个股详情页看到完整的技术+基本面信息
**从而** 做出综合判断，决定是否进一步研究或加入自选股

**验收标准：**
- [ ] 页面布局（参考东方财富，去广告简化版）：
  - **顶部**：股票名称 + 代码 + 实时价格 + 涨跌额/涨跌幅 + 今日成交量/成交额
  - **中部**：K线图区（占页面主体）
  - **底部标签页**：基本信息 / 财务数据 / 同行对比 / 相关分析
- [ ] K线图区支持4种周期切换：分时 / 日K / 周K / 月K，默认显示日K
- [ ] K线图指标：默认开启 MA（5/10/20日均线）+ 成交量柱，支持在指标栏切换开关 MACD / KDJ
- [ ] 底部"基本信息"标签：所属申万一级行业 + 二级行业、总市值、流通市值、上市日期
- [ ] 底部"财务数据"标签：PE（市盈率TTM）、PB（市净率）、最近一年营收、最近一年净利润、同比增长率（各指标旁用颜色标注正负：红色上涨/绿色下跌）
- [ ] 底部「**相关分析**」标签：展示模块一知识库中所有涉及该股的历史分析文章，具体规则：
  - 来源：AI分析「推荐关注股票」章节中明确出现该股代码或名称的文章
  - 展示内容：文章标题 + 摘要（≤80字）+ 保存日期，按日期倒序排列
  - 点击文章条目：跳转到模块一知识库，定位并展开该篇文章
  - 无相关文章时：显示"暂无相关分析"（不显示引导按钮）
  - 文章数量徽标：标签名旁显示文章数量，如「相关分析 (3)」；为0时显示「相关分析」不显示数字
- [ ] 页面右上角有「加入自选股」按钮，点击选择分组后添加

### Story 3：搜索股票

**作为** 散户
**我希望** 直接输入股票名称或代码搜索
**从而** 快速找到想看的股票

**验收标准：**
- [ ] 模块二顶部导航栏有常驻搜索框，支持输入股票代码（如"300750"）或股票名称（如"宁德时代"）
- [ ] 输入2个字符后实时展示下拉匹配列表，每项显示：股票名称 + 代码 + 今日涨跌幅
- [ ] 点击搜索结果直接进入个股详情页
- [ ] 搜索支持模糊匹配（输入"宁德"能找到"宁德时代"）
- [ ] 搜索无结果时显示"未找到相关股票，请检查代码或名称"

### Story 4：自选股管理

**作为** 散户
**我希望** 将关注的股票分组管理
**从而** 快速区分重仓股、观察股等不同状态的股票

**验收标准：**
- [ ] 默认创建3个分组：**重仓股** / **观察股** / **备选股**
- [ ] 支持新增自定义分组（分组名最长10字）
- [ ] 支持对现有分组改名（不能删除"默认"分组，可删除自定义分组）
- [ ] 删除分组时，若分组内有股票，提示"该分组有X只股票，删除后将移入「观察股」"
- [ ] 自选股列表每行展示：股票名称 + 代码 + 当前价格 + 今日涨跌幅 + 所属行业 + 模块三缠论信号灯（绿/黄/红，模块三未接入时显示"-"）
- [ ] 点击任意自选股跳转个股详情页
- [ ] 支持从自选股列表直接移除某只股票

---

### Story 5：行业导航与行业内股票对比

**作为** 散户
**我希望** 进入某个行业，看到该行业所有股票的对比数据
**从而** 横向比较，找到行业内估值更低/业绩更好的标的

**验收标准：**
- [ ] 模块二左侧有固定的申万一级行业导航菜单（31个行业，只显示名称）
- [ ] 点击某个行业，主区域展示该行业股票综合对比表，包含以下列：
  - 股票名称 + 代码
  - 今日涨跌幅（颜色标注）
  - PE（市盈率TTM）
  - PB（市净率）
  - 近一年营收（亿元）
  - 近一年净利润（亿元）
  - 净利润同比增长率
- [ ] 对比表支持按任意列排序（点击列标题升序/降序切换）
- [ ] 默认按今日涨跌幅降序排列
- [ ] 点击任意股票行，跳转到该股个股详情页
- [ ] 行业对比表顶部显示行业今日整体涨跌幅（行业内所有股票涨跌幅均值）

## 功能需求

### 核心功能

**功能 1：K线图渲染引擎**
- 技术方案：使用 **ECharts 5** 实现K线图，这是最成熟的中文生态图表库
- 支持4种周期：分时（1分钟聚合）/ 日K / 周K / 月K
- 内置指标：
  - **均线**：MA5（白）/ MA10（黄）/ MA20（紫），默认全开
  - **成交量**：柱状图，涨日红色，跌日绿色，默认显示
  - **MACD**：DIF线 + DEA线 + MACD柱，默认关闭，可在指标区点击开启
  - **KDJ**：K线 + D线 + J线，默认关闭，可在指标区点击开启
- K线图支持鼠标拖动平移、滚轮缩放
- 鼠标悬停时显示当根K线的OHLCV数据（开/高/低/收/量）
- **K线图是整个模块最重要的功能，渲染必须正确，优先级最高**

**功能 2：股票数据获取（AKShare集成）**
- 数据来源：**AKShare**（免费、开源A股数据库）
- 日K/周K/月K：拉取历史数据缓存到PostgreSQL，首次请求全量拉取（3年），后续每日增量更新
- 分时数据：当天交易时段内每15分钟从AKShare刷新一次，非交易时段不请求
- 财务数据：PE/PB/营收/净利润，每日收盘后更新一次
- **数据延迟15-30分钟可接受，不需要实时WebSocket**
- Redis缓存：日K数据缓存24小时，分时数据缓存15分钟

**功能 3：自选股分组存储**
- 分组和股票列表存储在PostgreSQL，与用户账号关联
- 自选股支持跨设备同步（登录同一账号即可）
- 每个用户最多创建10个分组，每组最多100只股票

**功能 4：文章-股票关联表**
- 描述：保存分析文章时，从「推荐关注股票」章节自动提取股票代码，写入关联表
- 数据库：`article_stock_relations`表（article_id, stock_code, stock_name）
- 用途：支撑模块一「股票视图」和模块二「相关分析」标签的数据查询
- 提取时机：用户点击「保存到知识库」确认后，后端解析正文自动写入
- 边界：若章节不存在或无股票，则关联表中无该文章的记录，不报错

**功能 5：申万行业股票映射**
- 静态数据文件：将所有A股按申万一级行业预先分类，存储为PostgreSQL表
- 数据来源：AKShare的`stock_board_industry_name_em`接口，开发前一次性初始化
- 每月更新一次（新股加入行业分类）

## 范围外（本期不做）

- **K线画线工具**（趋势线、压力位标注）—— 用户明确不需要
- **Level 2数据**（逐笔成交、大单追踪）—— 超出免费数据范围
- **股票财务深度分析**（杜邦分析、现金流等）—— 用户财务知识有限，不需要
- **港股/美股** —— 只做A股
- **实时WebSocket推送** —— 数据延迟可接受，不上实时方案
- **K线指标自定义**（参数调整）—— 用默认参数即可
- **复权方式切换**（前复权/后复权/不复权）—— MVP默认前复权

## 技术约束

### 性能
- K线图首次渲染：≤2秒（历史数据从本地PostgreSQL读取，不实时调用AKShare）
- 从模块一点击股票代码到侧边栏K线展示：≤1.5秒
- 行业对比表加载：≤3秒（数据每日缓存，不实时计算）
- 分时图刷新间隔：15分钟（交易时段）

### 数据
- AKShare调用限制：单IP每分钟请求数有限制，通过本地缓存规避
- 历史数据存储：预计每只股票3年日K约750条记录，1000只股票约75万条，PostgreSQL完全可承载
- 分时数据不持久化（每天数据量大，只保留当天）

### 集成
- **模块一联动**：文章内股票代码点击 → 侧边栏展示K线（前端路由参数传递股票代码）
- **模块三联动**：自选股列表中的缠论信号灯数据由模块三提供（`/api/strategy/{code}/signal`接口）
- **全局搜索**：顶部搜索框为全局组件，在模块一/二/三中均可使用

### 技术栈
- 前端：React 18 + TypeScript + Ant Design 5 + **ECharts 5**（K线图）
- 后端：Python FastAPI
- 数据库：PostgreSQL（历史K线 + 财务数据 + 自选股）+ Redis（缓存）
- 数据源：AKShare

## 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| AKShare接口变更或限流 | 中 | 高 | 本地缓存所有历史数据；封装统一数据服务层，便于切换数据源 |
| ECharts K线渲染显示异常 | 低 | 高 | 开发时优先跑通K线渲染Demo；使用成熟的ECharts K线配置模板 |
| 分时数据格式不统一 | 中 | 中 | 分时图做专项数据清洗和格式统一处理 |
| 行业股票数量过多导致对比表卡顿 | 低 | 中 | 行业对比表默认分页（每页20条），虚拟滚动优化 |
| 财务数据更新不及时 | 低 | 低 | 财务数据标注"数据截止日期"，用户知晓即可 |
