## 执行摘要

面向A股散户的AI事件分析与个人投研知识库模块，解决两大核心痛点：
1. **"看到新闻，不知道影响哪只股票"**：AI按六段式结构生成完整分析，事件类型自动匹配最优提示词
2. **"分析认知无处沉淀"**：每次分析可保存归档，按申万行业分类形成可全文检索的个人知识库

用户每天1-2小时投入，主要在早上开盘前和晚上收盘后使用。相比同花顺/东财等工具，本模块的差异化在于：**结构化分析 + 知识沉淀 + 跨模块联动**。

---

## 问题陈述

**当前痛点（按严重程度）**：

1. **认知无法沉淀**：分析完毕后结论散落在各工具中，无法系统回顾和复盘成长
2. **新闻到标的的转化断层**：知道某行业受影响，但找不到具体哪只股票值得关注
3. **追高被套**：缺乏判断当前是否为合适入场时机的参考依据
4. **信息来源分散**：4个工具（同花顺/东财/雪球/微信公众号）来回切，同一信息重复确认浪费时间

**解决方案**：
- AI代替用户完成基础研究，输出按类型优化的结构化分析报告
- 报告自动归档到申万行业，形成可全文检索的个人知识库
- 大事提醒机制确保不错过关键时间节点

**预期价值**：
- 每次看到市场事件，当天可得到结构化分析，而非模糊感受
- 3个月后形成完整行业研究记录，能看到自己认知积累和进步

## 成功指标

**核心 KPI：**
- 知识库积累：每月保存分析文章 ≥ 10 篇
- 行业覆盖：3个月内知识库覆盖申万行业 ≥ 15个
- 使用持续性：每周使用 ≥ 3次
- 搜索使用：上线后2周内，用户使用搜索功能 ≥ 5次（验证知识库价值）

**验证方式**：后台统计文章保存数量、行业分布、活跃频率、搜索次数

## 用户故事 & 验收标准

### Story 1：新用户引导

**作为** 第一次使用的散户
**我希望** 打开系统后立即知道怎么用
**从而** 不需要看任何说明书就能上手

**验收标准：**
- [ ] 首次登录后，输入框下方显示3个预置示例问题（点击可直接运行）
- [ ] 示例覆盖三种类型：地缘政治类 / 政策法规类 / 财报季报类
- [ ] 点击任意示例，自动填入输入框并触发分析，用户全程看完整流程
- [ ] 跑完一次后，示例引导隐藏，不再干扰日常使用

### Story 2：AI 事件分析

**作为** 普通散户
**我希望** 选择事件类型并输入描述，让AI帮我分析对A股的影响
**从而** 快速知道应该关注哪些行业和股票

**验收标准：**
- [ ] 输入框上方显示**5个**类型切换按钮：「地缘政治」「政策法规」「财报季报」「**产业链分析**」「其他」，默认选中「其他」
- [ ] 用户输入问题后点击「分析」，系统根据选中类型调用对应提示词模板
- [ ] 采用**流式输出**（SSE），AI返回的文字实时逐段显示，而非等待全部完成后一次性出现
- [ ] 流式输出开始后 ≤ 3秒内显示第一段文字；如果3秒内没有任何内容，显示"AI正在思考中..."
- [ ] 选择「产业链分析」时，分析结果包含**七个章节**（比其他类型多一章）：事件背景 / 影响逻辑 / **产业链传导表** / 受益行业 / 受损行业 / 推荐关注股票 / 风险提示
- [ ] 其余四种类型（地缘政治/政策法规/财报季报/其他）分析结果包含六个固定章节：事件背景 / 影响逻辑 / 受益行业 / 受损行业 / 推荐关注股票 / 风险提示
- [ ] 推荐关注股票：每个受益行业最多列出3只代表性股票，避免信息过载
- [ ] 受益行业和受损行业从申万31个一级行业中选取
- [ ] 分析任务在**后台运行**，用户可自由切换到知识库或其他页面，不影响分析进行
- [ ] 分析进行中，页面顶部显示"分析进行中..."常驻状态条；完成后状态条变为"分析完成，点击查看"
- [ ] 分析进行中「分析」按钮变为灰色不可点击，防止重复提交
- [ ] 用户输入超过20字后，内容自动保存为本地草稿（localStorage）；刷新页面后草稿自动恢复
- [ ] AI调用失败自动重试2次（间隔5秒），重试失败后显示错误原因和手动重试按钮；草稿输入内容保留不丢失
- [ ] 输入为空时提示"请输入事件描述"；不足10字时提示"描述太简短，请详细说明"

### Story 3：保存到行业知识库

**作为** 普通散户
**我希望** 看完分析后自主决定是否保存
**从而** 只留下有价值的内容，不产生噪音

**验收标准：**
- [ ] AI生成分析的同时，同步生成一个**≤15字的中文标题**（如"美伊冲突对能源军工行业影响"），作为文章在知识库中的显示标题
- [ ] AI生成分析的同时，同步生成一段**≤80字的中文摘要**：**2句话，结论导向**：第1句说明"谁受益/谁受损"，第2句说明"最大风险是什么"（示例："能源和军工板块短期受益，油价上涨传导至化工成本。最大风险是地缘冲突快速降温导致行情反转。"）
- [ ] 分析结果顶部展示自动生成的标题，用户可点击直接编辑修改
- [ ] 分析结果标题下方展示自动生成的摘要，用户可点击直接编辑修改
- [ ] 分析结果底部显示「保存到知识库」和「不保存」两个按钮
- [ ] 点击保存前，展示AI识别的行业标签列表，支持用户手动增删；若用户删除全部标签，提示"请至少保留1个行业，或选择「未分类」"
- [ ] 确认后写入数据库，标题、摘要、正文、行业标签一并保存
- [ ] 保存成功显示提示"已保存，关联了X个行业"
- [ ] 不保存直接关闭，不写入数据库
- [ ] 降级规则：若LLM未输出摘要，截取正文"推荐关注股票"章节前80字作为摘要，保证字段不为空

### Story 5：相似问题检测

**作为** 已积累了一定数量分析文章的散户
**我希望** 输入新问题时系统提示我历史上分析过类似问题
**从而** 避免重复分析，也能对比自己认知的变化

**验收标准：**
- [ ] 用户在输入框输入问题后（停止输入超过1秒），后台自动检索知识库中的相似文章
- [ ] 如检索到相似度较高的历史文章（标题/内容关键词重叠度高），在输入框下方显示提示卡："你3个月前分析过类似问题 →《XXX》，要对比吗？"
- [ ] 点击提示卡可查看历史文章，不影响当前输入内容
- [ ] 用户可忽略提示继续分析，系统不阻断流程
- [ ] 知识库文章数量为0时不触发检测（无历史数据无法检索）
- [ ] 相似检测基于**标题 + 摘要 + 行业标签**的关键词重叠匹配，不调用AI（不产生额外费用）；有了摘要后匹配维度更丰富，比纯标题匹配准确度更高

## 功能需求

### 核心功能

**功能 1：提示词模板系统**
- 描述：根据用户选择的事件类型，在调用LLM时注入不同的分析框架提示词
- **五类模板**：
  - 地缘政治：重点分析军工/能源/贵金属/航运等受影响行业
  - 政策法规：重点分析政策直接覆盖行业、上下游传导链
  - 财报季报：重点分析个股业绩超预期/不及预期对行业估值的影响
  - **产业链分析**：以事件影响起点行业为锚，逐层追踪上下游传导（最多5层），每层含影响方向/影响程度/时间节奏/推荐关注股票
  - 其他：通用六段式分析框架
- 边界情况：模板切换不影响用户已输入的问题内容

**功能 2：AI 分析引擎（含流式输出）**
- 描述：接收问题+模板类型，调用配置的大模型，通过**SSE（Server-Sent Events）**流式返回分析内容
- 用户流程：选类型 → 输入问题 → 点击分析 → 文字实时流式显示 → 展示结果 → 确认行业标签 → 保存/丢弃
- **流式输出机制**：
  - 后端使用 `StreamingResponse` 推送内容块，前端 EventSource 接收
  - 分析任务在后台进行，用户可切换页面，任务不中断
  - 任务完成后状态栏出现"分析完成，点击查看"通知
- **标题 + 摘要同步生成**：分析完成时，LLM在同一次调用中同步输出：
  - `TITLE:`：≤15字中文标题
  - `SUMMARY:`：≤80字结论导向摘要（2句话：受益/受损 + 最大风险）
  - 两个字段零额外API调用，仅多消耗约50个token
- 错误处理：
  - 网络超时：自动重试2次，间隔5秒；草稿输入内容保留
  - API Key错误/额度不足：显示"AI服务配置有误，请检查系统设置"
  - 返回格式异常：降级展示原始内容，标注"格式解析异常，显示原始内容"

**功能 3：申万行业标签提取 & 确认**
- 描述：AI从分析内容中提取行业，从申万31个一级行业列表中选择，保存前展示给用户确认
- **产业链分析类型的特殊规则**：
  - 自动提取传导表中所有层级涉及的行业（可能多达5-10个行业）
  - 保存确认弹窗中，按传导层级分组展示行业标签（如"第一层：石油石化" / "第二层：基础化工"）
  - 用户可逐个勾选/取消，默认全部选中
  - 提示文案改为"本次产业链分析涉及X个行业，请确认要关联的行业"
- 边界情况：若事件与所有行业关联度低，允许行业列表为空

**功能 4：知识库全文搜索**
- 描述：对所有已保存文章提供全文检索，支持中文分词
- 搜索范围：文章标题 + 正文内容 + 行业标签 + 文章内出现的股票代码
- 技术方案：PostgreSQL + pg_trgm 扩展 或 后端分词后建立倒排索引
- 边界情况：搜索无结果时显示"未找到相关文章"，不报错

**功能 5：新用户引导**
- 描述：首次登录时展示3个可点击的示例问题，引导用户跑通完整流程
- 示例题目：
  1. （地缘政治）"美国对伊朗实施新一轮制裁，对A股有什么影响？"
  2. （政策法规）"国家出台新能源汽车补贴政策，哪些行业会受益？"
  3. （财报季报）"茅台季报净利润超预期20%，对白酒行业有什么影响？"
- 触发条件：系统检测该用户保存文章数量为0时展示；已保存过文章则不展示

**功能 6：相似问题检测**
- 描述：用户输入问题时，实时检索知识库，发现相似历史分析则给出提示
- 触发时机：用户停止输入超过1秒后触发检索
- 匹配逻辑：对用户输入和历史文章标题/行业标签进行关键词重叠匹配，不调用LLM
- 展示方式：输入框下方非阻断式提示卡，用户可忽略
- 边界情况：知识库为空 → 不触发；找不到相似文章 → 不显示提示，静默

**功能 7：大事提醒**
- 描述：手动添加未来重要事件，系统提前3天和当天提醒
- 提醒方式：站内通知（必须）+ 邮件通知（可选，Phase 2实现）

### 范围外（本期不做）

- **分析卡片化折叠展示**（Phase 2）：默认折叠只看摘要+受益行业+推荐股票
- **事件类型自动识别**（Phase 2）：AI自动判断类型，用户无需手动选
- **晚间复盘今日摘要**（Phase 2）：今日保存文章+市场事件+自选股涨跌汇总
- **多人协作**（不做）
- **AI自动抓取新闻**（不做）
- **分析命中率追踪**（不做，用户明确不需要）

## 技术约束

### 性能
- **流式输出首字延迟**：≤3秒内显示第一段文字
- AI分析完整响应：目标60秒内，最长不超过120秒（用户体验因流式输出大幅改善）
- 超过3秒无响应则显示"AI正在思考中..."骨架屏
- 知识库列表首屏加载 ≤ 2秒（最多分页20条）
- 搜索响应 ≤ 1秒（PostgreSQL全文搜索索引）

### 安全
- 公网部署，必须登录认证（账号密码）
- 初期单用户，预留多用户扩展接口
- LLM API Key存储在服务器端环境变量，不暴露给前端

### 集成

| 集成点 | 方式 | 说明 |
|--------|------|------|
| 大模型接口 | OpenAI兼容格式 | 支持DeepSeek/OpenAI/其他，界面可配置Base URL和Key |
| 申万行业数据 | 本地维护 | 31个一级行业静态数据，注入提示词 |
| 全文搜索 | PostgreSQL内置 | pg_trgm扩展实现中文模糊搜索 |
| 模块二联动 | 跳转链接 | 文章内股票代码点击跳转，接口预留 |

### 技术栈
- 前端：React 18 + Ant Design 5，PC优先
- 后端：Python FastAPI
- 数据库：PostgreSQL（文章、行业日志、提醒事项、全文搜索索引）
- 部署：Docker Compose，云服务器
