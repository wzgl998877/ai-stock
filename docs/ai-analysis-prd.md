## 执行摘要

面向普通 A 股散户的 AI 宏观事件分析模块，解决投资者"看到新闻不知道影响哪些股票"的核心痛点。用户输入任意宏观事件，AI 自动生成六段式完整分析报告，覆盖事件背景、影响逻辑、受益行业、受损行业、推荐股票以及风险提示。

报告按申万行业分类自动归档，形成可检索的个人投资知识库，帮助用户积累长期投研经验。同时提供大事提醒功能，对于提前已知的重要时间节点（如财报季、重要会议），系统提前提醒并支持一键触发 AI 分析。

---

## 问题陈述

**当前痛点**：
- 散户每天看到大量财经新闻，但不具备专业能力快速判断哪些行业和股票会受到影响
- 即使花时间研究，也没有系统记录，事后复盘找不到当时的分析依据
- 错过重要事件窗口期，等反应过来市场已经提前定价

**解决方案**：
- AI 代替投资者完成基础研究，输出结构化分析报告
- 报告自动归档到行业维度，形成可检索的个人知识库
- 大事提醒机制确保用户不错过关键时间节点

**预期价值**：
- 每次看到市场事件，当天即可得到系统化分析，而不是模糊感受
- 1-3 个月使用后，形成完整的行业研究记录，辅助长期投资决策

### 主要用户：普通 A 股散户
- **角色**：无专业金融背景的个人投资者
- **目标**：看到市场热点，快速判断有没有投资机会；积累自己的投研知识

## 用户故事 & 验收标准

### Story 1：AI 事件分析

**作为** 普通散户
**我希望** 输入一段市场事件描述，让 AI 帮我分析对 A 股的影响
**从而** 快速知道应该关注哪些行业和股票

**验收标准：**
- [ ] 用户在输入框输入问题，点击"分析"按钮后，系统开始调用 AI
- [ ] 1-2 分钟内返回分析结果，超过 30 秒需显示进度提示
- [ ] 分析结果包含六个固定章节：事件背景 / 影响逻辑 / 受益行业 / 受损行业 / 推荐关注股票 / 风险提示
- [ ] 受益行业和受损行业均从申万行业分类（104 个三级行业）中选取
- [ ] AI 调用失败时自动重试 1-2 次，重试失败后显示明确错误信息并提供手动重试按钮

### Story 2：保存分析文章

**作为** 普通散户
**我希望** 在看完分析后自主决定是否保存
**从而** 只留下我认为有价值的内容，不产生垃圾数据

**验收标准：**
- [ ] 文章底部显示"保存"和"不保存"两个按钮
- [ ] 点击"保存"后，文章自动关联到文章中提到的所有申万行业
- [ ] 保存成功后显示提示，并展示关联了哪些行业
- [ ] 不保存则直接关闭，不写入数据库

### Story 3：行业日志查看

**作为** 普通散户
**我希望** 查看某个行业的历史分析记录
**从而** 了解这个行业被分析了多少次，有什么历史规律

**验收标准：**
- [ ] 支持两种视图切换：「行业视图」（按行业分组展示）和「时间线视图」（按时间倒序展示）
- [ ] 行业视图：左侧显示行业列表（有文章记录的行业），右侧显示该行业下的文章列表
- [ ] 时间线视图：所有已保存文章按日期倒序排列，显示标题、日期、关联行业标签
- [ ] 点击任意文章可展开查看完整内容
- [ ] 文章关联的股票代码可点击，跳转到模块二的个股详情页（未来联动）

### Story 4：大事提醒

**作为** 普通散户
**我希望** 提前记录重要的市场事件（如财报季、美联储会议）
**从而** 事件临近时系统提醒我，不会错过分析窗口期

**验收标准：**
- [ ] 支持手动添加提醒：填写事件名称、日期、关联行业（可选）
- [ ] 提前 3 天系统发出站内提醒（页面右上角通知标记）
- [ ] 事件当天弹窗提醒，支持一键触发对该事件的 AI 分析
- [ ] 已过期事件自动归档，不再显示在主列表

## 功能需求

### 核心功能

**功能 1：AI 分析引擎**
- 描述：接收用户输入的自然语言问题，调用配置的大模型（DeepSeek/OpenAI 等），按六段式结构返回分析内容
- 用户流程：输入问题 → 点击分析 → 显示加载动画 → 展示结果 → 用户选择保存或丢弃
- 边界情况：
  - 输入为空时，提示"请输入事件描述"
  - 输入字数不足 10 字，提示"描述太简短，请详细说明"
  - AI 返回内容格式不符合预期时，展示原始内容并标注"格式解析失败"
- 错误处理：
  - 网络超时：自动重试 2 次，间隔 5 秒
  - API Key 错误/额度不足：显示"AI 服务配置有误，请检查设置"

**功能 2：行业标签自动提取**
- 描述：AI 在生成受益行业和受损行业时，从申万 104 个三级行业分类列表中选择匹配项，而非自由发挥
- 实现方式：系统提示词中注入完整的申万行业列表，要求模型只从列表中选择
- 边界情况：若事件与所有行业关联度极低，允许行业列表为空

**功能 3：文章归档与检索**
- 描述：已保存文章存入数据库，支持按行业、按时间筛选
- 边界情况：
  - 同一事件多次分析，作为独立文章分开存储
  - 文章最大存储数量不限（个人知识库，按需增长）

**功能 4：大事提醒**
- 描述：用户可添加未来重要事件，系统在提前 3 天和事件当天发出提醒
- 提醒方式：站内通知（必须）+ 邮件通知（可选，需配置邮箱）

### 范围外（本期不做）
- 多人协作编辑同一篇分析
- AI 自动从财经新闻网站抓取事件（自动模式）
- 分析文章的社交分享功能（暂不考虑，后续版本支持多用户后再做）
- 股票价格历史与分析文章的自动关联验证

## 技术约束

### 性能
- AI 分析响应时间：目标 60 秒内，最长不超过 120 秒
- 超过 10 秒时前端显示进度条/等待动画
- 页面加载速度：日志列表首屏 ≤ 2 秒

### 安全
- 系统部署到公网服务器，必须有登录认证
- 初期支持单用户（账号密码登录），预留多用户扩展能力
- API Key（LLM 密钥）存储在服务器端环境变量，不暴露给前端

### 集成
- **大模型接口**：通过界面配置 API Base URL 和 API Key，兼容 OpenAI 格式（DeepSeek、OpenAI 等均支持）
- **申万行业分类**：本地维护一份申万三级行业名单（104 个行业），作为 AI 提示词约束
- **模块间联动**：文章中的股票代码需与模块二的个股页面建立跳转链接（联动接口预留，V2 实现）

### 技术栈
- 前端：React 18 + Ant Design 5
- 后端：Python FastAPI
- 数据库：PostgreSQL（存储文章、行业日志、提醒事项）
- 部署：Docker Compose，部署到云服务器

## MVP 范围 & 阶段规划

### Phase 1：MVP（初始版本，必须完成）
- 输入问题 → AI 生成六段式分析 → 选择保存
- 文章自动关联申万行业标签
- 行业日志：行业视图 + 时间线视图切换
- 大事提醒：添加、提前 3 天提醒、当天弹窗
- 点击文章中的股票跳转到模块二（模块二完成后联通）

**MVP 定义**：用户能完整走通"输入事件 → 看到分析 → 保存到行业 → 事后检索"的完整流程。

### Phase 2：增强（上线后迭代）
- LLM 配置界面（支持用户自行填写 API Key 和模型名）
- 大事提醒支持邮件通知
- 文章关键词搜索功能
- 多用户注册与登录

### 未来考虑
- AI 自动抓取财经新闻触发分析
- 分析命中率追踪（事后验证 AI 预测是否准确）
- 与模块三的缠论数据联动（分析看多的股票同时显示缠论信号）

## 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| LLM 返回格式不稳定，六段结构解析失败 | 中 | 高 | 提示词严格约束输出格式，解析失败时降级显示原始文本 |
| AI 分析内容质量差（行业/股票不准确） | 中 | 中 | 产品定位为"辅助参考"，文章页面加投资风险提示 |
| 申万行业 AI 识别偏差（选错行业） | 低 | 中 | 保存前展示行业标签供用户确认，支持手动修改 |
| API Key 费用超出预期 | 低 | 中 | 界面显示本月已调用次数估算；DeepSeek 成本远低于 OpenAI |
| 数据库文章量增大后查询变慢 | 低 | 低 | 行业字段加索引，日期字段加索引，按需分页加载 |
